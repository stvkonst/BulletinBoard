Создать приложение Доска объявлений.
Автор(1) может иметь несколько объявлений(2) в разных рубриках(3).
Приложение должно позволять:

1) Создавать:
- Рубрику(3) с названием и с/без объявлений(2).
- Объявление(2)(название, дата публикации, текст объявления, стоимость услуги,
  автор(1)(телефоны(4)(отдельная сущность), адрес(5)(отдельная сущность), имя, э/почта(6)(отдельная сущность)).

Автор(1) состоит из:
Имя
телефон(4)
адрес(5)
эл.почта(6)

2) Редактировать объявление(2), личную информацию автора(1) и рубрику(3).
3) Удалять объявление(2)/рубрику(3).
4) Удалять все объявления автора.
5) Показывать все объявления из одной рубрики или нескольких рубрик.
6) Фильтрации(объявления(2) автора(1), по ключевому слову в названии, по дате публикации).

!!Стоп!!
7) Выводить определенное количество объявлений на странице(pagination).
8) Подписывать автора на объявления с заданными параметрами.
9) Удалять неактивные объявление(Scheduling).

К 8 пункту создать коллекцию доменов с полями:
Category
String title
Price from
Price to
Author


В приложении использовать Spring 5, JPA.
		      **** Технические требования ****
Приложение должно:
- быть максимально плотно покрыто тестами.
- содержать четкое логическое разделение между представлением, логикой и источником данных.
- cборка проекта/запуск тестов должна производится средствами maven.
- готовое приложение должно быть доступно на GitHub репозитории.
- все ключевые моменты логики приложения должны содержать поясняющие JavaDocs.
- обязательно должна присутствовать валидация данных.
