Создать приложение Доска объявлений. [Автор] может иметь несколько [Объявлений] в разных [Рубриках].
Приложение должно позволять:

* 1) Создавать [Рубрику] с названием.
* 2) Создавать [Автора]: имя, [Телефон], [Адрес], [Имейл].
* 3) Создавать [Объявление] (название, дата публикации, текст объявления, стоимость услуги, [Автор], [Рубрика].

* 4) Редактировать [Объявление].
* 5) Редактировать [Автора].
* 6) Редактировать [Рубрику].

* 7) Удалять одно [Объявление].
* 8) Удалять [Рубрику] вместе со всеми ее [Объявлениями].
* 9) Удалять все [Объявления] [Автора].
* 10) Удалять [Автора] и вместе все его [Объявления].

* 11) Показывать все [Объявления] из одной [Рубрики] или нескольких [Рубрик].
* 12) Фильтрации [Объявления] по айди [Автора], по ключевому слову в названии, по дате публикации. !!!

* 13) Выводить определенное количество объявлений на странице(pagination). SpringData
* 14) Подписывать автора на объявления с заданными параметрами.
* 15) Удалять неактивные объявление(Scheduling). CRON!!!

К 14 пункту создать коллекцию доменов с полями:
Category
String title
Price from
Price to
Author


В приложении использовать Spring 5, JPA.
		      **** Технические требования ****

- быть максимально плотно покрыто тестами.
- *содержать четкое логическое разделение между представлением, логикой и источником данных.
- *cборка проекта/запуск тестов должна производится средствами maven.
- *готовое приложение должно быть доступно на GitHub репозитории.
- все ключевые моменты логики приложения должны содержать поясняющие JavaDocs.
- обязательно должна присутствовать валидация данных.
